Design history Land summary (v1.1)

Second iteration of a prototype that re-presents the existing land summary customer journey using the Defra interactive mapping component (opens in new tab) (v0.0.22-alpha).

We continued to re-present the existing land customer journey, introducing an historical view for two land parcels.

Design review

We shared the first iteration of the prototype with the wider team. This gave the team the opportunity to review and give feedback on the designs. We also collated all the business requirements, user needs and pain points from a previous workshop.

The design review resulted in the removal of the specific date search functionality. Research had indicated it confused users. We made the assumption that users would come to the service knowing the land parcel they were looking for.

Historical view of land parcels

Because no historical data is available from the public API, we obtained historical data for two land parcels from the LMS team. The two land parcel IDs are: SX3963 2340 and SX3964 7821.

We included a link to the historical view of the land parcel.

We included some inset copy highlighting the dates for which the historical view was relevant. We included a link for a user to navigate to the current version of the land parcel.

UI iterations

Header

The header was redesigned to align it with the rest of the service. The sub navigation was removed. The phase banner introduced. The business name, SBI, customer name and sign out link added.

Map component updates

Hedgerows

We re-presented the existing highlighting of hedgerow data. When a user clicks on a hedgerow on the map the corresponding hedgerow data is highlighted in data column.

Layers

We re-presented the existing layer toggle functionality. The user has the ability to toggle on / off the following layers: Land covers, Land parcel outlines and Hedgerows.

Demo

To enable us to design a layout that works for different quantities of land parcels and business sizes we built out a demo that allows us to load any individual SBI data in the view land summary prototype.

Another benefit of producing the demo is that it can be used during research to enable participants to view their own land during research sessions.

Choosing land cover colours

We built a colour tuning tool into the prototype so we could try different colour combinations, for land parcel boundries and land covers, live on the map during design sessions. Adding a colours flag to the URL.

Each land cover type on the map is drawn in its own colour. We grouped the palette greens for grassland and woodland, blues for water (drains, rivers, ponds), greys for hard surfaces and roads, and warmer tones for buildings and farmyards.

  • Each map element and land cover type has its own colour picker, plus a slider for the hover tint, and changes apply to the map instantly.
  • A chosen palette is remembered in the browser, so it persists between page loads.
  • Copy share link produces a URL that reproduces the exact palette, so a combination can be shared with the team for feedback without changing any code.
  • Copy palette exports the values as JSON, ready to paste in as the new default once a palette is agreed.
