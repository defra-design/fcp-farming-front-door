Design history Land summary (v1.0)

First iteration of a prototype that re-presents the existing land summary customer journey using the Defra interactive mapping component (opens in new tab) (v0.0.22-alpha). A full-bleed map and sidebar layout, and a side-by-side alternative layout.

Existing customer journey

The current View land journey on the Rural Payments Service shows users a list of their land parcels and a separate map. This iteration brings the data and the map together in a single view.

  • Screenshots of the existing land summary customer journey are in Mural (opens in new tab).

Existing user research

An iteration of the existing journey was designed and tested in Mural before the availability of the interactive mapping component.

  • Screens in Mural (opens in new tab).
  • User research report on Confluence (opens in new tab)

New customer journey

Two pages have been prototyped. Both share the same fullscreen two-column layout. GOV.UK header and service navigation, with the remaining viewport split between a scrolling sidebar (one third) and the map (two thirds).

  • Land summary (/v18/view-land). Date input as the entry point, summary stats (number of parcels, total area), and a sortable table of parcels. Clicking a row in the table or a land parcel on the map highlights the parcel in the other.
  • Parcel details (/v18/view-land-parcel?parcel=SK82871468). A single parcel with its land covers (count, area, totals) and hedgerows (count, length, totals). Each land cover is rendered as a distinct dataset on the map.

Layout

On desktop the layout is two columns. On screens narrower than 640px the map column collapses to zero, the sidebar takes the full viewport, and a "Map view" button (relocated into the sidebar) opens the map fullscreen.

Alternative desktop layouts

Two layouts have been prototyped. They show identical data using identical components. The only differences are the column order.

  • Default (view-land / view-land-parcel). Sidebar on the left, map on the right.
  • Alternative (view-land-alt / view-land-parcel-alt). Map on the left, sidebar on the right.
  • Mobile."View map" button to toggle between data and map views.

Interactive mapping component plugins used

  • scaleBarPlugin for metric scale bar.
  • mapStylesPlugin for three map styles: Default (Liberty), Satellite, Light (Positron).
  • interactPlugin for single land parcel selection on the map, deselect on click outside.
  • datasetsMaplibrePlugin used on the view-land-parcel map to render each land cover type as its own labelled dataset, plus the parcel outline on top.

Customised UI

  • Fill patterns on the view-land-parcel map (cross-hatch, dot, horizontal hatch, vertical hatch) layered over base colours so that visually similar land covers can be told apart, and so the map remains legible if printed in black and white.
  • Exit button on the view-land-parcel map so a user can return to the land summary from fullscreen.
  • "Map view" button relocation on mobile, the component's open-map button is moved into the sidebar so it sits among the other controls.
  • Fullscreen chrome handling when the map enters fullscreen, the GOV.UK header and service navigation are hidden.
  • Table row styling pointer cursor, hover background and a selected state on the summary parcels table, kept in sync with map selection.

Data for prototyping

All data is fetched at runtime from the publicly available RPA WFS endpoints on environment.data.gov.uk (opens in new tab). No data is held by the prototype. A single SBI 106332870 is hardcoded for now.

  • RPA:LandParcels for boundaries and area for every parcel on the holding.
  • RPA:LandCovers for land cover polygons within each parcel, with a description and area.
  • RPA:HedgeControl for hedgerows associated with a parcel, with a length.

Limitations

  • SBI is hardcoded.
  • The date input and WFS query is not yet time-bounded.
  • Last updated 1 January 2010 is a placeholder.
  • Hedgerows are listed in the sidebar but not drawn on the map.
  • No error or loading messages.
  • Colour contrast on land cover fills.

Next steps
